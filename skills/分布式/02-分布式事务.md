# 分布式事务

## 问题
如何保证跨服务的数据一致性？

## 解决方案

### 1. 2PC (两阶段提交)

```
Coordinator                    Participants
    │                              │
    │────── Prepare ──────────────▶│
    │◀───── Vote Commit/Abort ────│
    │                              │
    │────── Commit/Abort ─────────▶│
    │◀────── Ack ─────────────────│
```

### 2. TCC (Try-Confirm-Cancel)

```java
public interface AccountTccService {
    @TryMethod
    void tryDeduct(String userId, BigDecimal amount);
    
    @ConfirmMethod
    void confirmDeduct(String userId, BigDecimal amount);
    
    @CancelMethod
    void cancelDeduct(String userId, BigDecimal amount);
}
```

### 3. 消息事务

```java
// 本地消息表
@Transactional
public void trans() {
    // 1. 业务操作
    orderService.create(order);
    // 2. 保存消息
    messageService.save(new Message("扣减库存", order.getId()));
}
// 异步投递消息
```

## 最佳实践

- 强一致性: 2PC / 3PC
- 最终一致性: TCC / Seata AT
- 消息队列: RocketMQ 事务消息
- 根据业务选择合适方案
