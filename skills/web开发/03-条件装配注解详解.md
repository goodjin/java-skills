# Spring Boot 条件装配注解详解

## 概述

Spring Boot 条件装配是其自动配置的基础，通过各种 `@Conditional*` 注解，可以根据不同的条件决定是否创建或跳过某些 Bean。Spring Boot 提供了丰富的条件装配注解，满足各种场景的需求。

## 源码分析

### 1. 条件装配基础

Spring Boot 的条件装配基于 Spring Framework 的 `@Conditional` 注解，定义了一系列组合注解：

```java
// Spring Framework 基础注解
@Target({ElementType.TYPE, ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Conditional(ContextConfiguration.LoadTimeWeaverConfiguration.WeaverAware.class)
public @interface Conditional {
    Class<? extends Condition>[] value();
}

// Spring Boot 扩展注解
@Target({ElementType.TYPE, ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Conditional(OnClassCondition.class)
public @interface ConditionalOnClass {
    Class<?>[] value() default {};
    String[] name() default {};
}
```

### 2. 条件评估器基类

```java
// SpringBootCondition.java
public abstract class SpringBootCondition implements Condition {
    
    @Override
    public final boolean matches(ConditionContext context, AnnotatedTypeMetadata metadata) {
        // 获取条件类型
        String conditionKind = getConditionKind(metadata);
        
        // 评估条件
        ConditionOutcome outcome = getMatchOutcome(context, metadata);
        
        // 记录日志
        logConditionOutcome(metadata, outcome);
        
        return outcome.isMatch();
    }
    
    protected abstract ConditionOutcome getMatchOutcome(ConditionContext context, 
        AnnotatedTypeMetadata metadata);
}
```

### 3. OnClassCondition 实现

```java
// OnClassCondition.java
@Order(Ordered.HIGHEST_PRECEDENCE)
class OnClassCondition extends FilteringSpringBootCondition {
    
    @Override
    public ConditionOutcome getMatchOutcome(ConditionContext context, AnnotatedTypeMetadata metadata) {
        ClassLoader classLoader = context.getClassLoader();
        ConditionMessage matchMessage = ConditionMessage.empty();
        
        // 检查 @ConditionalOnClass
        List<String> onClasses = getCandidates(metadata, ConditionalOnClass.class);
        if (onClasses != null) {
            List<String> missing = filter(onClasses, ClassNameFilter.MISSING, classLoader);
            if (!missing.isEmpty()) {
                return ConditionOutcome.noMatch(
                    ConditionMessage.forCondition(ConditionalOnClass.class)
                        .didNotFind("required class", "required classes")
                        .items(Style.QUOTE, missing)
                );
            }
        }
        
        // 检查 @ConditionalOnMissingClass
        List<String> onMissingClasses = getCandidates(metadata, ConditionalOnMissingClass.class);
        if (onMissingClasses != null) {
            List<String> present = filter(onMissingClasses, ClassNameFilter.PRESENT, classLoader);
            if (!present.isEmpty()) {
                return ConditionOutcome.noMatch(
                    ConditionMessage.forCondition(ConditionalOnMissingClass.class)
                        .found("unwanted class", "unwanted classes")
                        .items(Style.QUOTE, present)
                );
            }
        }
        
        return ConditionOutcome.match(matchMessage);
    }
}
```

### 4. OnBeanCondition 实现

```java
// OnBeanCondition.java
@Order(Ordered.LOWEST_PRECEDENCE)
class OnBeanCondition extends FilteringSpringBootCondition {
    
    @Override
    public ConditionOutcome getMatchOutcome(ConditionContext context, AnnotatedTypeMetadata metadata) {
        boolean shouldSkip = false;
        
        // 获取 @ConditionalOnBean 注解属性
        Map<String, Object> attributes = metadata.getAnnotationAttributes(
            ConditionalOnBean.class.getName(), true);
        
        if (attributes != null) {
            Class<?>[] value = (Class<?>[]) attributes.get("value");
            String[] name = (String[]) attributes.get("name");
            SearchStrategy searchStrategy = (SearchStrategy) attributes.get("search");
            
            // 检查 Bean 是否存在
            if (!checkForBeans(context, value, name, searchStrategy)) {
                shouldSkip = true;
            }
        }
        
        return shouldSkip ? 
            ConditionOutcome.noMatch("required bean not found") : 
            ConditionOutcome.match();
    }
}
```

## 核心条件注解

### @ConditionalOnClass / @ConditionalOnMissingClass

根据类路径是否存在某个类来决定是否创建 Bean：

```java
// 存在指定类时才创建
@ConditionalOnClass(DataSource.class)
@Configuration
public class DataSourceAutoConfiguration {
    // ...
}

// 不存在指定类时才创建
@ConditionalOnMissingClass("com.example.MyClass")
@Configuration
public class MyAutoConfiguration {
    // ...
}
```

### @ConditionalOnBean / @ConditionalOnMissingBean

根据 Bean 是否存在来决定是否创建 Bean：

```java
// 存在指定 Bean 时才创建
@ConditionalOnBean(DataSource.class)
@Configuration
public class DataSourceTransactionManagerAutoConfiguration {
    @Bean
    public DataSourceTransactionManager dataSourceTransactionManager(DataSource dataSource) {
        return new DataSourceTransactionManager(dataSource);
    }
}

// 不存在指定 Bean 时才创建
@ConditionalOnMissingBean
@Bean
public MyService myService() {
    return new MyService();
}
```

### @ConditionalOnProperty

根据配置属性来决定是否创建 Bean：

```java
// 配置属性匹配
@ConditionalOnProperty(name = "spring.datasource.enabled", havingValue = "true")
@Configuration
public class DataSourceAutoConfiguration {
    // ...
}

// 配置不存在时也创建
@ConditionalOnProperty(name = "feature.enabled", havingValue = "true", matchIfMissing = true)
@Configuration
public class FeatureAutoConfiguration {
    // ...
}

// 多个属性
@ConditionalOnProperty(name = {"feature.a", "feature.b"}, havingValue = "true")
@Configuration
public class MyAutoConfiguration {
    // ...
}
```

### @ConditionalOnWebApplication / @ConditionalOnNotWebApplication

根据是否是 Web 应用来决定：

```java
// 是 Web 应用时创建
@ConditionalOnWebApplication
@Configuration
public class WebMvcAutoConfiguration {
    // ...
}

// 不是 Web 应用时创建
@ConditionalOnNotWebApplication
@Configuration
public class NonWebAutoConfiguration {
    // ...
}
```

### @ConditionalOnWarDeployment / @ConditionalOnNotWarDeployment

根据是否部署在 WAR 容器中：

```java
// 部署在 WAR 容器时创建
@ConditionalOnWarDeployment
@Configuration
public class WarDeploymentAutoConfiguration {
    // ...
}

// 非 WAR 部署时创建
@ConditionalOnNotWarDeployment
@Configuration
public class SpringBootDeployAutoConfiguration {
    // ...
}
```

### @ConditionalOnExpression

使用 SpEL 表达式作为条件：

```java
// SpEL 表达式匹配
@ConditionalOnExpression("${spring.datasource.enabled:true} and '${spring profiles.active}' != 'prod'")
@Configuration
public class MyAutoConfiguration {
    // ...
}

// 更复杂的表达式
@ConditionalOnExpression("#{'${my.config.values}'.split(',').contains('value1')}")
@Configuration
public class MyAutoConfiguration {
    // ...
}
```

### @ConditionalOnJava

根据 Java 版本：

```java
// Java 11 或更高版本
@ConditionalOnJava(JavaVersion.ELEVEN)
@Configuration
public class Java11AutoConfiguration {
    // ...
}

// Java 8 到 11（含）
@ConditionalOnJava(range = ConditionalOnJava.Range.EIGHT_TO_ELEVEN)
@Configuration
public class MyAutoConfiguration {
    // ...
}
```

### @ConditionalOnResource

根据资源文件是否存在：

```java
// 资源文件存在时
@ConditionalOnResource(resources = "classpath:db/migration/V1__init.sql")
@Configuration
public class FlywayAutoConfiguration {
    // ...
}
```

### @ConditionalOnSingleCandidate

当只有一个候选 Bean 时才创建：

```java
// 只有一个 DataSource Bean 时创建
@ConditionalOnSingleCandidate(DataSource.class)
@Configuration
public class DataSourcePoolMetadataAutoConfiguration {
    // ...
}
```

## 条件装配流程图

```
┌─────────────────────────────────────────────────────────────────┐
│                        条件评估流程                               │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  1. 扫描所有 @Conditional* 注解                                 │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  2. 按 @Order 排序条件                                          │
│     - 大多数条件使用 Ordered.LOWEST_PRECEDENCE                  │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  3. 依次评估每个条件                                            │
│     - 获取条件所需的上下文信息                                   │

│     - 记录匹配结果                                             │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  4. 汇总结果                                                    │
│     - 所有条件都匹配 → 创建 Bean                                │
│     - 任一条件不匹配 → 跳过 Bean                               │
└─────────────────────────────────────────────────────────────────┘
```

## 代码示例

### 1. 基本用法

```java
// 条件配置类
@Configuration
@ConditionalOnClass(DataSource.class)
@ConditionalOnProperty(name = "spring.datasource.enabled", havingValue = "true", matchIfMissing = true)
public class DataSourceAutoConfiguration {
    
    @Bean
    @ConditionalOnMissingBean
    public DataSource dataSource(DataSourceProperties properties) {
        return properties.initializeDataSourceBuilder().build();
    }
}
```

### 2. 组合多个条件

```java
@Configuration
// 所有条件都必须满足
@ConditionalOnClass(DataSource.class)
@ConditionalOnBean(DataSource.class)
@ConditionalOnProperty(name = "spring.datasource.hikari.enabled", havingValue = "true", matchIfMissing = true)
public class HikariDataSourceConfiguration {
    
    @Bean
    @Primary
    @ConditionalOnMissingBean
    public DataSource dataSource(DataSourceProperties properties) {
        HikariDataSource dataSource = properties.initializeDataSourceBuilder()
            .type(HikariDataSource.class)
            .build();
        return dataSource;
    }
}
```

### 3. 条件与配置属性

```java
// 配置属性类
@ConfigurationProperties(prefix = "my.feature")
public class MyFeatureProperties {
    private boolean enabled = false;
    private String name = "default";
    // getters/setters
}

// 条件使用配置属性
@Configuration
@EnableConfigurationProperties(MyFeatureProperties.class)
@ConditionalOnProperty(prefix = "my.feature", name = "enabled", havingValue = "true")
public class MyFeatureAutoConfiguration {
    
    @Autowired
    private MyFeatureProperties properties;
    
    @Bean
    @ConditionalOnMissingBean
    public MyFeatureService myFeatureService() {
        return new MyFeatureService(properties.getName());
    }
}
```

### 4. 自定义条件

```java
// 自定义条件注解
@Target({ElementType.TYPE, ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@Conditional(OnMyCustomCondition.class)
public @interface ConditionalOnMyCustom {
    String value() default "";
}

// 自定义条件实现
public class OnMyCustomCondition implements Condition {
    
    @Override
    public boolean matches(ConditionContext context, AnnotatedTypeMetadata metadata) {
        Map<String, Object> attributes = metadata.getAnnotationAttributes(
            ConditionalOnMyCustom.class.getName());
        
        if (attributes != null) {
            String value = (String) attributes.get("value");
            // 自定义匹配逻辑
            return "expected".equals(value);
        }
        
        return false;
    }
}

// 使用自定义条件
@Configuration
@ConditionalOnMyCustom("expected")
public class MyAutoConfiguration {
    // ...
}
```

### 5. 条件与 Aware 接口

```java
// 在条件中使用 BeanFactory
public class OnBeanCondition extends FilteringSpringBootCondition {
    
    @Override
    public ConditionOutcome getMatchOutcome(ConditionContext context, AnnotatedTypeMetadata metadata) {
        BeanFactory beanFactory = context.getBeanFactory();
        
        // 检查 Bean 是否存在
        boolean beanExists = beanFactory.containsBean("myBean");
        
        return beanExists ? 
            ConditionOutcome.match() : 
            ConditionOutcome.noMatch("myBean not found");
    }
}
```

### 6. 条件在方法级别

```java
@Configuration
public class MyConfiguration {
    
    // 默认 Bean
    @Bean
    @ConditionalOnMissingBean
    public MyService defaultMyService() {
        return new DefaultMyService();
    }
    
    // 仅当存在特定配置时创建
    @Bean
    @ConditionalOnProperty(name = "my.service.type", havingValue = "custom")
    @ConditionalOnMissingBean
    public MyService customMyService() {
        return new CustomMyService();
    }
}
```

## 最佳实践

### 1. 条件优先级

```java
// 优先级从高到低：
// 1. @ConditionalOnProperty - 通常最先评估
// 2. @ConditionalOnClass - 类加载时评估
// 3. @ConditionalOnMissingClass
// 4. @ConditionalOnBean - 需要 BeanFactory
// 5. @ConditionalOnMissingBean
// 6. @ConditionalOnSingleCandidate
```

### 2. 避免条件冲突

```java
// 好的实践：明确指定条件
@Configuration
@ConditionalOnClass(DataSource.class)  // 先检查类是否存在
@ConditionalOnBean(DataSource.class)     // 再检查 Bean 是否存在
public class DataSourceAutoConfiguration {
    // ...
}
```

### 3. 使用 @ConditionalOnMissingBean

```java
// 确保用户可以覆盖默认配置
@Bean
@ConditionalOnMissingBean
public MyService myService() {
    return new MyService();
}

// 用户可以定义自己的 MyService 来覆盖
@Configuration
public class MyCustomConfiguration {
    @Bean
    public MyService myService() {
        return new CustomMyService(); // 优先使用
    }
}
```

### 4. 条件评估日志

## 总结

Spring Boot 条件装配机制通过丰富的 `@Conditional*` 注解组合，实现了灵活、智能的 Bean 创建控制：

1. **类路径条件**: `@ConditionalOnClass`, `@ConditionalOnMissingClass`
2. **Bean 条件**: `@ConditionalOnBean`, `@ConditionalOnMissingBean`, `@ConditionalOnSingleCandidate`
3. **属性条件**: `@ConditionalOnProperty`
4. **环境条件**: `@ConditionalOnWebApplication`, `@ConditionalOnWarDeployment`
5. **表达式条件**: `@ConditionalOnExpression`
6. **版本条件**: `@ConditionalOnJava`
7. **资源条件**: `@ConditionalOnResource`

理解这些条件注解的使用场景和优先级，能够帮助开发者编写更加灵活、可配置的自动配置类。
