# Spring Boot 自动配置机制详解

## 概述

Spring Boot 自动配置（Auto Configuration）是 Spring Boot 框架最核心的特性之一，它能够根据类路径中的依赖、配置文件中的属性等因素，自动配置 Spring 应用所需的 Bean，从而极大简化了开发者的配置工作。

## 源码分析

### 1. 自动配置入口：@EnableAutoConfiguration

```java
// EnableAutoConfiguration.java
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Inherited
@AutoConfigurationPackage
@Import(AutoConfigurationImportSelector.class)
public @interface EnableAutoConfiguration {
    
    // 排除特定自动配置类
    Class<?>[] exclude() default {};
    
    // 通过名称排除自动配置类
    String[] excludeName() default {};
}
```

### 2. 自动配置选择器：AutoConfigurationImportSelector

```java
// AutoConfigurationImportSelector.java
public class AutoConfigurationImportSelector implements DeferredImportSelector, 
    BeanClassLoaderAware, ResourceLoaderAware, BeanFactoryAware, EnvironmentAware, Ordered {
    
    @Override
    public String[] selectImports(AnnotationMetadata annotationMetadata) {
        if (!isEnabled(annotationMetadata)) {
            return NO_IMPORTS;
        }
        AutoConfigurationEntry autoConfigurationEntry = getAutoConfigurationEntry(annotationMetadata);
        return StringUtils.toStringArray(autoConfigurationEntry.getConfigurations());
    }
    
    protected AutoConfigurationEntry getAutoConfigurationEntry(AnnotationMetadata annotationMetadata) {
        if (!isEnabled(annotationMetadata)) {
            return EMPTY_ENTRY;
        }
        
        // 1. 获取注解属性
        AnnotationAttributes attributes = getAttributes(annotationMetadata);
        
        // 2. 获取候选自动配置类
        List<String> configurations = getCandidateConfigurations(annotationMetadata, attributes);
        
        // 3. 去重
        configurations = removeDuplicates(configurations);
        
        // 4. 获取排除项
        Set<String> exclusions = getExclusions(annotationMetadata, attributes);
        
        // 5. 检查排除项
        checkExcludedClasses(configurations, exclusions);
        
        // 6. 移除排除的类
        configurations.removeAll(exclusions);
        
        // 7. 条件过滤
        configurations = getConfigurationClassFilter().filter(configurations);
        
        // 8. 触发自动配置导入事件
        fireAutoConfigurationImportEvents(configurations, exclusions);
        
        return new AutoConfigurationEntry(configurations, exclusions);
    }
    
    // 获取候选自动配置类
    protected List<String> getCandidateConfigurations(AnnotationMetadata metadata,
            AnnotationAttributes attributes) {
        ImportCandidates importCandidates = ImportCandidates.load(
            this.autoConfigurationAnnotation, getBeanClassLoader());
        List<String> configurations = importCandidates.getCandidates();
        return configurations;
    }
}
```

### 5. 自动配置排序

```java
// AutoConfigurationGroup.java
private static final class AutoConfigurationGroup 
    implements DeferredImportSelector.Group, BeanClassLoaderAware, BeanFactoryAware, ResourceLoaderAware {
    
    @Override
    public Iterable<Entry> selectImports() {
        // 排序自动配置类
        return sortAutoConfigurations(processedConfigurations, getAutoConfigurationMetadata()).stream()
            .map(this::getEntry)
            .toList();
    }
    
    private List<String> sortAutoConfigurations(Set<String> configurations,
            AutoConfigurationMetadata autoConfigurationMetadata) {
        return new AutoConfigurationSorter(getMetadataReaderFactory(), 
            autoConfigurationMetadata, this.autoConfigurationReplacements::replace)
            .getInPriorityOrder(configurations);
    }
}
```

## 核心类解释

### @EnableAutoConfiguration

- **作用**: 启用 Spring Boot 自动配置
- **原理**: 通过 `@Import` 导入 `AutoConfigurationImportSelector`
- **注意**: `@SpringBootApplication` 已包含此注解

### AutoConfigurationImportSelector

- **作用**: 选择要导入的自动配置类
- **实现**: `DeferredImportSelector` 接口
- **关键方法**: `selectImports()`、`getAutoConfigurationEntry()`

### ImportCandidates

- **作用**: 加载 `META-INF/spring/*.imports` 文件中的配置类
- **实现**: 使用 `SpringFactoriesLoader` 加载

### AutoConfigurationImportFilter

- **作用**: 过滤自动配置类
- **默认实现**: `OnClassCondition`（检查类是否存在）

### SpringBootCondition

- **作用**: 所有条件装配的基类
- **功能**: 提供条件评估的统一接口

## 代码示例

### 1. 定义自动配置类

```java
// 方式一：使用 @AutoConfiguration
@AutoConfiguration
@ConditionalOnClass(DataSource.class)
@ConditionalOnProperty(name = "spring.datasource.enabled", havingValue = "true", matchIfMissing = true)
public class DataSourceAutoConfiguration {
    
    @Bean
    @ConditionalOnMissingBean
    public DataSource dataSource(DataSourceProperties properties) {
        return properties.initializeDataSourceBuilder().build();
    }
}

// 方式二：使用 @Configuration（传统方式）
@Configuration
@ConditionalOnClass(DataSource.class)
public class DataSourceAutoConfiguration {
    // ...
}
```

### 2. 条件依赖

```java
@AutoConfiguration
@ConditionalOnClass(DataSource.class)
@ConditionalOnProperty(name = "spring.datasource.enabled", havingValue = "true", matchIfMissing = true)
public class DataSourceAutoConfiguration {
    
    // 依赖于 DataSource
    @Configuration
    @ConditionalOnBean(DataSource.class)
    @ConditionalOnMissingBean
    public static class DataSourceTransactionManagerConfiguration {
        @Bean
        public DataSourceTransactionManager dataSourceTransactionManager(DataSource dataSource) {
            return new DataSourceTransactionManager(dataSource);
        }
    }
}
```

### 3. 排除自动配置

```java
// 方式一：注解排除
@SpringBootApplication(exclude = {DataSourceAutoConfiguration.class})
public class MyApplication {
    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
    }
}

// 方式二：配置排除
@SpringBootApplication
public class MyApplication {
    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, "--spring.autoconfigure.exclude=org.springframework.boot.autoconfigure.jdbc.DataSourceAutoConfiguration");
    }
}

// 方式三：application.yml 排除
spring:
  autoconfigure:
    exclude:
      - org.springframework.boot.autoconfigure.jdbc.DataSourceAutoConfiguration
```

### 4. 查看自动配置报告

```java
// 方式一：通过日志
// 在 application.yml 中配置
logging:
  level:
    org.springframework.boot.autoconfigure: DEBUG

// 方式二：通过 Web 端点
// 引入 spring-boot-starter-actuator
management:
  endpoints:
    web:
      exposure:
        include: conditions
```

访问 `/actuator/conditions` 端点可以查看所有自动配置类的评估报告。

### 5. 自定义自动配置导入顺序

```java
@AutoConfiguration
// 在 DataSourceAutoConfiguration 之后配置
@AutoConfigureAfter(DataSourceAutoConfiguration.class)
public class JpaAutoConfiguration {
    // ...
}

// 或者指定顺序
@AutoConfigureOrder(Ordered.HIGHEST_PRECEDENCE + 10)
@AutoConfiguration
public class MyAutoConfiguration {
    // ...
}
```

### 1. 自动配置类命名规范

### 2. 使用 @ConditionalOnMissingBean

```java

    return new MyService();
```

```java

@Retention(RetentionPolicy.RUNTIME)
@AutoConfiguration
@ConditionalOnClass(RedisTemplate.class)
@ConditionalOnProperty(name = "spring.redis.enabled", havingValue = "true", matchIfMissing = true)
public @interface EnableRedisAutoConfiguration {
}

// 使用
@EnableRedisAutoConfiguration
public class MyApplication {
}
```

### 4. 属性前缀规范

```java

    private String url;
    private String username;
    private String password;
    // getters/setters
}

// 使用
@DataSourceProperties
@AutoConfiguration

    private DataSourceProperties properties;

```

```java
// 条件优先级从高到低：
```

### 6. 调试自动配置

```java
// 在启动类中添加
@SpringBootApplication
public class MyApplication {
    public static void main(String[] args) {
        SpringApplication app = new SpringApplication(MyApplication.class);
        // 开启自动配置报告
        app.setAddCommandLineProperties(true);
        ConfigurableApplicationContext context = app.run(args);
        
        // 输出自动配置报告
        if (context instanceof AbstractApplicationContext abstractContext) {
            String[] beans = context.getBeanNamesForType(AutoConfiguration.class);
            System.out.println("Auto-configured beans: " + Arrays.toString(beans));
        }
    }
}
```

## 总结

Spring Boot 自动配置机制是其核心优势之一，通过以下方式实现：

1. **约定优于配置**: 使用约定俗成的命名和位置
2. **条件装配**: 通过各种 `@Conditional*` 注解实现灵活的装配条件
3. **SPI 机制**: 使用 `SpringFactoriesLoader` 加载自动配置
4. **延迟导入**: 使用 `DeferredImportSelector` 实现延迟导入
5. **智能排序**: 使用 `@AutoConfigureBefore/After` 控制加载顺序

理解自动配置机制对于深入掌握 Spring Boot、解决配置问题、自定义 Starter 都非常重要。
