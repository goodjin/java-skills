# Spring Boot 嵌入式服务器原理详解

## 概述

Spring Boot 支持多种嵌入式服务器，包括 Tomcat、Jetty、Reactor Netty、Undertow 等。框架能够根据类路径中的依赖自动选择合适的服务器，无需开发者手动配置。



### 1## 源码分析. Web 应用类型推断

```java
// WebApplicationType.java
public enum WebApplicationType {
    NONE,    // 非 Web 应用
    SERVLET, // Servlet Web 应用
    REACTIVE;// 响应式 Web 应用
    
    // 推断 Web 应用类型
    public static WebApplicationType deduce() {
        for (Deducer deducer : SpringFactoriesLoader.forDefaultResourceLocation()
                .load(Deducer.class)) {
            WebApplicationType deduced = deducer.deduceWebApplicationType();
            if (deduced != null) {
                return deduced;
            }
        }
        return isServletApplication() ? WebApplicationType.SERVLET : WebApplicationType.NONE;
    }
    
    private static boolean isServletApplication() {
        String[] servletIndicatorClasses = {
            "jakarta.servlet.Servlet",
            "org.springframework.web.context.ConfigurableWebApplicationContext"
        };
        for (String indicator : servletIndicatorClasses) {
            if (!ClassUtils.isPresent(indicator, null)) {
                return false;
            }
        }
        return true;
    }
}
```

### 2. ApplicationContextFactory

```java
// ApplicationContextFactory.java
public interface ApplicationContextFactory {
    
    ApplicationContextFactory DEFAULT = new DefaultApplicationContextFactory();
    
    // 根据 WebApplicationType 创建上下文
    ConfigurableApplicationContext create(WebApplicationType webApplicationType);
}

// DefaultApplicationContextFactory.java
class DefaultApplicationContextFactory implements ApplicationContextFactory {
    
    @Override
    public ConfigurableApplicationContext create(WebApplicationType webApplicationType) {
        return getFromSpringFactories(webApplicationType, 
            ApplicationContextFactory::create,
            this::createDefaultApplicationContext);
    }
    
    // 从 SpringFactories 加载特定的 ApplicationContextFactory
    private <T> T getFromSpringFactories(WebApplicationType webApplicationType,
            BiFunction<ApplicationContextFactory, WebApplicationType, T> action,
            Supplier<T> defaultResult) {
        for (ApplicationContextFactory candidate : 
                SpringFactoriesLoader.loadFactories(ApplicationContextFactory.class, 
                    getClass().getClassLoader())) {
            T result = action.apply(candidate, webApplicationType);
            if (result != null) {
                return result;
            }
        }
        return defaultResult.get();
    }
}
```

### 4. Web 服务器启动

```java
// ServletWebServerApplicationContext.java
public class ServletWebServerApplicationContext extends GenericWebApplicationContext 
    implements WebServerApplicationContext {
    
    private volatile WebServer webServer;
    
    @Override
    protected void onRefresh() {
        super.onRefresh();
        // 创建并启动 Web 服务器
        this.webServer = createWebServer();
        this.webServer.start();
    }
    
    private WebServer createWebServer() {
        ServletWebServerFactory factory = getWebServerFactory();
        WebServer webServer = factory.getWebServer();
        // 注册 Servlet 上下文
        if (servletContext != null) {
            servletContext.addServlet("dispatcherServlet", new DispatcherServlet(webApplicationContext))
                .addMapping("/");
        }
        return webServer;
    }
}
```

### 5. Reactive Web 服务器

```java
// ReactiveWebServerApplicationContext.java
public class ReactiveWebServerApplicationContext extends GenericReactiveWebApplicationContext 
    implements WebServerApplicationContext {
    
    private volatile WebServer webServer;
    
    @Override
    protected void onRefresh() {
        super.onRefresh();
        this.webServer = createWebServer();
        this.webServer.start();
    }
    
    private WebServer createWebServer() {
        ReactiveWebServerFactory factory = getReactiveWebServerFactory();
        return factory.getWebServer();
    }
}
```

## 服务器选择机制

```
┌─────────────────────────────────────────────────────────────────┐
│                 Web 应用类型推断流程                              │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  1. 检查类路径                                                  │

│     - org.springframework.web.reactive.DispatcherHandler       │

                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  2. 创建对应的 ApplicationContext                               │
│     - Servlet → AnnotationConfigServletWebApplicationContext  │

└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  3. 根据 ApplicationContext 类型                                │

└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  4. 创建嵌入式服务器                                            │
│     - Servlet: Tomcat / Jetty / Undertow                       │
│     - Reactive: Netty / Reactor Netty / Tomcat / Jetty         │
└─────────────────────────────────────────────────────────────────┘
```

## 核心类解释

### WebApplicationType

- **作用**: 枚举 Web 应用类型
- **取值**: NONE, SERVLET, REACTIVE
- **推断依据**: 类路径中是否存在 Servlet/Reactive 相关类

### WebServer

- **接口**: 嵌入式 Web 服务器统一接口
- **方法**: `start()`, `stop()`, `getPort()`

### ServletWebServerFactory

- **作用**: 创建 Servlet Web 服务器的工厂接口
- **实现**: TomcatServletWebServerFactory, JettyServletWebServerFactory, UndertowServletWebServerFactory

### ReactiveWebServerFactory

- **作用**: 创建 Reactive Web 服务器的工厂接口
- **实现**: NettyReactiveWebServerFactory, JettyReactiveWebServerFactory, TomcatReactiveWebServerFactory

### WebServerApplicationContext

- **作用**: Web 服务器应用上下文接口
- **实现**: ServletWebServerApplicationContext, ReactiveWebServerApplicationContext

## 最佳实践

### 1. 服务器调优

```java
// Tomcat 调优
@Bean
public TomcatServletWebServerFactory tomcatServletWebServerFactory() {
    TomcatServletWebServerFactory factory = new TomcatServletWebServerFactory();
    
    // 线程池配置
    factory.addConnectorCustomizers(connector -> {
        ProtocolHandler handler = connector.getProtocolHandler();
        if (handler instanceof Http11NioProtocolHandler nioHandler) {
            nioHandler.setMaxThreads(200);
            nioHandler.setMinSpareThreads(10);
            nioHandler.setAcceptCount(100);
            nioHandler.setMaxConnections(10000);
        }
    });
    
    // 压缩配置
    factory.addConnectorCustomizers(connector -> {
        connector.setProperty("compression", "on");
        connector.setProperty("compressionMinSize", "1024");
        connector.setProperty("noCompressionUserAgents", "gozilla, traviata");
        connector.setProperty("compressibleMimeType", "text/html,text/xml,text/plain,text/css,text/javascript,application/javascript,application/json");
    });
    
    return factory;
}
```

### 2. 健康检查

```java
// 继承 WebServerApplicationContext
@Component
public class ServerHealthIndicator implements HealthIndicator {
    
    private final WebServerApplicationContext context;
    
    public ServerHealthIndicator(WebServerApplicationContext context) {
        this.context = context;
    }
    
    @Override
    public Health health() {
        try {
            WebServer server = context.getWebServer();
            if (server.getPort() > 0) {
                return Health.up()
                    .withDetail("port", server.getPort())
                    .withDetail("type", server.getClass().getSimpleName())
                    .build();
            }
            return Health.down().build();
        } catch (Exception e) {
            return Health.down().withException(e).build();
        }
    }
}
```

## 总结

Spring Boot 嵌入式服务器机制的核心是：

1. **自动选择**: 根据类路径自动选择 Web 服务器
2. **统一接口**: 通过 `WebServerFactory` 抽象不同服务器
3. **自动配置**: 使用 `@ConditionalOnClass` 按需加载
4. **灵活配置**: 通过 `application.properties` 配置服务器参数
5. **可扩展**: 支持自定义服务器工厂

理解服务器选择和启动原理，有助于：
- 优化服务器配置
- 排查启动问题
- 实现高级定制
- 选择合适的服务器
